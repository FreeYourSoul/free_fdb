A nix utility is also present in order to install/build locally free\+\_\+fdb for local development. It is used by the CI.


\begin{DoxyCode}{0}
\DoxyCodeLine{nix-\/build . -\/A ffdb}
\end{DoxyCode}


If you want to integrate free\+\_\+fdb from your local nix project, you can add an attribute by fetching the recipe.\+nix file to your nix file directly with the url. Or you can just copy the recipe.\+nix file into your repository.

Example taken from the \href{https://github.com/FreeYourSoul/FyS}{\texttt{ FyS project}}, \{commit-\/hash\} has to be replaced in order to fix the recipe to use in your project\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{let pkgs = import <nixpkgs> \{ \};}
\DoxyCodeLine{in with pkgs; rec \{}
\DoxyCodeLine{}
\DoxyCodeLine{  \# External Dependencies (personal)}
\DoxyCodeLine{  ffdb = (callPackage (builtins.fetchurl}
\DoxyCodeLine{    "https://raw.githubusercontent.com/FreeYourSoul/free\_fdb/\{commit-\/hash\}/recipee.nix") \{}
\DoxyCodeLine{      rev = "\{commit-\/hash\}";}
\DoxyCodeLine{    \});}
\DoxyCodeLine{    }
\DoxyCodeLine{    \#.... your nix}
\end{DoxyCode}
